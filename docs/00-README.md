# REZNO Master Prompt

Before making any code changes:

1. Read every document inside the `/docs` directory.
2. Treat those documents as the official source of truth.
3. Understand the project before implementing anything.
4. Preserve existing functionality.
5. Never rewrite stable features without a valid reason.
6. Build production-quality code.
7. Follow the documented architecture, coding standards, and roadmap.
8. If documentation is missing, suggest improvements before implementation.
9. After reading all documentation, summarize your understanding, then continue development.
